## 3. 判断理由

### 3.1 0287 を done 化せず active 継続とする理由

SGK-2026-0288 / 0289 / 0290 により、当初の抽出阻害要因だった ReconPipeline 自己参照、state access protocol 不在、plan/replan hotspot は解消した。現在の未達は機能退行や抽出方針ではなく、完了 gate のサイズ条件（facade 800-1500 行、最大単一ファイル 1500 行）のみである。

1. **same-file helper 化の限界**: hotspot の構造改善は進んだが、抽出先 helper が facade 内に留まるため net 行数削減になりにくい
2. **compat bridge 残存**: `ConductorState` 導入後も `__new__` 系テスト互換のため react/state access compat helper が残っている
3. **次段は module/service 外出しが必要**: これ以上の純減には facade 内 helper 化ではなく、coordinator/service の別 module 退避が必要

### 3.2 Monkeypatch caller migration を Phase 2-5 に先送りした理由

ユーザーが「段階的 compat alias」を選択。shim に compat alias を維持しつつ、Phase 2-5 で段階的にテストパッチを `master_conductor_facade` ターゲットへ移行する戦略。
