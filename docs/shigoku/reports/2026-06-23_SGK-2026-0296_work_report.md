# 作業報告書：Strict Entry Gate Review Fixes

## 1. 実装概要

SGK-2026-0295 の実装後レビューで指摘された 7 件の問題を修正した。

## 2. 修正内容

### Fix 1: EntryGateFacade キャッシュ粒度 (Finding High)
**問題**: `run_once()` がプロセス全体で最初の結果を返し続け、別ターゲット・別 credential の再チェックがスキップされた。
**修正**: `run_once()` をグローバルキャッシュからコンテキストキーキャッシュに変更。キー = `target|goal|profile|resume_session_id|auth_presence`。異なるコンテキストでは必ず再評価。
**ファイル**: `src/core/preflight/entry_gate.py`

### Fix 2: ターゲット正規化 (Finding High)
**問題**: `shigoku --target example.com` のような scheme なし入力が preflight で AUTH_UNKNOWN になり停止。
**修正**: `_normalize_target_for_preflight()` ヘルパーを追加し、scheme なし入力に `https://` を補完。
**ファイル**: `src/main.py`

### Fix 3: ToolUpdatePolicy 配線 (Finding High)
**問題**: `ToolUpdatePolicy` は実装されているが `ToolChecker` から呼ばれておらず dead code 化。
**修正**: `ToolChecker.__init__` に `update_policy` パラメータを追加。`_check_single_tool()` から policy を呼び出し。managed tool は BinaryManager 経由で install/update。
**ファイル**: `src/core/preflight/tool_check.py`

### Fix 4: Goal 導出 (Finding High)
**問題**: `goal=str(mode)` で `bugbounty` が渡され、bbot の `required_for_goals=["recon","full"]` に一致せずチェック漏れ。
**修正**: `_derive_goal_for_preflight(args)` で CLI アクションから実行目的を導出。`--target` → `recon`、`--crawl` → `crawl` など。demo/projects/report/rag は preflight 対象外に。
**ファイル**: `src/main.py`

### Fix 5: 公開ターゲットの認証プローブ緩和 (Finding Medium)
**問題**: Cookie/Bearer が不要な公開 recon でも authenticated marker 不在で strict failure。
**修正**: `AuthProbe` に `auth_required` フラグ追加。認証情報あり時のみ AUTHENTICATED 判定必須。認証情報なし時は login/challenge/block 検出は行うが、UNKNOWN は pass。
**ファイル**: `src/core/preflight/auth_probe.py`, `src/core/preflight/entry_gate.py`

### Fix 6: Caido アイデンティティ確認強化 (Finding Medium)
**問題**: token 未設定時、8080 番に何か応答があれば Caido として通っていた。
**修正**: `_check_caido_identity()` 追加。`/graphql` エンドポイントの応答形、Caido 固有ヘッダー、ボディ内 "Caido" 文字列で確認。確認不能時は `CAIDO_IDENTITY_UNVERIFIED` で fail-close。
**ファイル**: `src/core/preflight/caido_check.py`

### Fix 7: InteractiveBridge Cookie パース + /resume コンテキスト抽出 (Finding Medium)
**問題**: Cookie 文字列が dict 化されず preflight に届かない。`/resume` で session の target が使われていない。
**修正**: `_parse_cookies()` ヘルパー追加（文字列→dict）。`cmd_resume` で session から target/mode/goal を抽出して PreflightContext 構築。target なし時は preflight skip。
**ファイル**: `src/core/conductor/interactive_bridge.py`, `src/cli/commands.py`
