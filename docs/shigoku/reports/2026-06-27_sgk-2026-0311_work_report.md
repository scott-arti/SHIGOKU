## 2. 実装判断

### Admission reject 設計
選択肢 (a) 例外 / (b) None / (c) rejected フラグ付き ParallelTask のうち、(c) を採用。
- `create_parallel_task()` で admission check 後に `admitted=False` + `reject_reason` を設定
- `execute_parallel()` で admitted=False の task をフィルタ（semaphore/executor slot 非消費）
- 呼び出し元（MasterConductor）修正ゼロ

### category → lane マッピング
`CATEGORY_TO_LANE` を `parallel_orchestrator.py` に定義。未知 category は `read_only` にフォールバック（安全側）。

### 最小影響遵守
- `ethics_guard.py` / `enhanced_ethics_guard.py` は一切変更なし
- `create_parallel_task()` の全追加パラメータは default 付き keyword-only
- `ParallelTask` への field 追加は末尾 default 付きのみ
- 既存 `src/config.py`（deprecated）は変更なし
