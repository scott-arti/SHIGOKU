# 作業完了報告書: SGK-2026-0312 Phase 3（dispatch context isolation / per-dispatch instance）

## 1. 成果概要
SwarmDispatcher が Swarm インスタンスを pool 再利用するのを廃止し、dispatch ごとに新規インスタンスを生成・dispatch 後に close する per-dispatch instance 方式を実装。同時 dispatch での `current_context` / `history` / findings / url_results / auth_headers / cookies 汚染を構造的に防止。

## 2. 変更内容
### 変更ファイル
- `src/core/engine/swarm_dispatcher.py` (単一ファイル)
- `tests/unit/engine/test_swarm_dispatcher_per_dispatch_isolation.py` (新規)

### 変更 A: `_get_or_create_swarm()` を per-dispatch 生成へ
- pool キャッシュ（L95-96 `_swarm_pool` からの取得）を削除
- pool への格納（L134 `self._swarm_pool[swarm_name] = swarm`）を削除
- 戻り値を `self._swarm_pool.get(swarm_name)` から `swarm` へ変更
- shared service injection（network/llm/event_bus/recipe/rag）は維持

### 変更 B: dispatch 経路3箇所に `try/finally close` を追加
- `dispatch()` for ループ: `swarm = None` 初期化 + `finally` で close
- `_dispatch_to_single_swarm()`: 既存 `finally`（aggressive_limiter.release）内に close を統合
- `dispatch_to_all()`: `swarm = None` 初期化 + `finally` で close

### テスト追加
- T-0.1: serial dispatch baseline（mock swarm で dispatch pipeline 回帰検証）
- T-3.1: per-dispatch distinct instance / no-leak
- T-1.1: shared service identity 保持
- T-2.1: concurrent dispatch findings isolation
- T-2.1b: barrier forced interleaving isolation
- T-2.4: concurrent dispatch history isolation
- T-2.5: exception close safety
- T-2.5b: shared network_client not closed
