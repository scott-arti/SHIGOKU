# SGK-2026-0328 作業完了報告書

## 1. 要約
- `--translate-logs` オプションと、その背後にあった実験的ログ翻訳機能を削除した。
- CLI 日本語化は既に本機能へ依存していなかったため、help と message catalog から obsolete 表示だけを外した。
- TDD で help 出力と message key の削除を先に失敗させ、最小差分で GREEN に戻した。

## 2. 変更内容
- `src/main.py`
  - `--translate-logs` の `argparse` 定義を削除
  - `args.translate_logs` による環境変数設定と `enable_log_translation()` 呼び出しを削除
- `src/cli/messages.py`
  - `argparse.translate_logs.help` キーを削除
- `src/core/utils/log_translator.py`
  - 未使用の実験的ログ翻訳ハンドラを削除
- `src/prompts/roles/tool_output_analysis.md`
  - `Used by` 注記から `log_translator` を削除
- `tests/test_cli_localization.py`
  - `--help` に `--translate-logs` が残っていないことを確認する回帰テストを追加
  - `argparse.translate_logs.help` が message catalog に存在しないことを確認する回帰テストを追加

## 3. 設計判断
- 本変更は「CLI 日本語化」と「内部ログ後翻訳」の責務分離を明確にする cleanup と位置づけ、新しい置換機能は入れなかった。
- `tool_output_analysis` role は `micro_agent` と `proxy_log_analyzer` で継続利用されているため、prompt テンプレートそのものは保持した。
- 歴史的な完了済み計画書・報告書は監査記録として残し、現行コードと task 文書だけを最新状態へ寄せた。
