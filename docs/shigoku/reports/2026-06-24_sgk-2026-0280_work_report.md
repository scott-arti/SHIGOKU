## 1. 完了した項目

### 1.1 イベントペイロード契約の確立 (Step 1-2)
- `src/core/agents/swarm/auth/reauth_contracts.py` を新設
- `SessionExpiredPayload`, `ReauthSuccessPayload`, `ReauthFailedPayload` の dataclass を定義
- 各 payload に `validate()` メソッドを実装し、必須フィールド欠落を検出
- `reauth_attempt_id`, `auth_context_version` をコードベースに導入
- Reason code taxonomy: `missing_refresh_url`, `missing_login_request`, `network_client_unavailable`, `csrf_token_missing`, `token_extraction_failed`, `login_replay_non_200`, `reauth_storm_suppressed`, `unsupported_auth_scheme` (8 codes)
- `AuthContext` dataclass: `login_request`, `refresh_url`, `cookie_jar_snapshot`, `last_auth_error`, `last_auth_status`, `reauth_triggered_at`, `reauth_completed_at`, `auth_context_version`

### 1.2 NetworkClient の SESSION_EXPIRED 発行更新 (Step 2)
- `src/core/infra/network_client.py`: 401検知時の payload に `reauth_attempt_id`, `origin_task_id`, `auth_context_version`, `request_headers` を追加
- `current_reauth_context` context var を新設（MasterConductor が設定）

### 1.3 AutoReauthSpecialist の実トークン抽出化 (Step 4)
- `src/core/agents/swarm/auth/reauth_specialist.py` を全面改修
- 合成トークン・プレースホルダーを完全除去
- 正規表現による `access_token`/`refresh_token` 抽出 (`_ACCESS_TOKEN_PATTERNS`, `_REFRESH_TOKEN_PATTERNS`)
- Login replay 前の CSRF token preflight (`_CSRF_TOKEN_PATTERNS`)
- NetworkClient 未設定時の `network_client_unavailable` reason code
- Refresh 失敗時の `missing_refresh_url`, `token_extraction_failed`, `login_replay_non_200` reason codes
- テスト: `tests/core/agents/swarm/auth/test_reauth_specialist.py` (32 tests pass)

### 1.4 ReauthOrchestrator の実装 (Step 3)
- `src/core/engine/reauth_orchestrator.py` を新設
- Single-flight: 同一 target + principal + auth_context_version に対して同時1件のみ許可
- Cooldown window: `REAUTH_FAILED` 後の再試行抑制
- Storm suppression: 短期間に5件超の SESSION_EXPIRED で再認証停止
- Degradation: 3回連続失敗でターゲットを degraded 状態に遷移
- Task quarantine: `REAUTH_FAILED` 時に auth-sensitive/stateful な pending タスクを隔離
- Resume Policy Matrix: `read-only`(ALLOW_RETRY), `stateful`(REQUIRE_STATE_CHECK), `auth-sensitive`(DISCARD)
- テスト: `tests/core/engine/test_reauth_orchestrator.py` (22 tests pass)

### 1.5 MasterConductor との統合 (Step 3)
- `REAUTH_FAILED` イベント購読を追加 (`_handle_reauth_failed`)
- `_handle_session_expired` に single-flight / cooldown / storm suppression / degradation チェックを追加
- `_handle_reauth_success` に quarantine release を追加
- `AuthContext` 管理 (`self._auth_ctx`) を追加
- Run ledger への reauth 失敗記録
- 統合テスト: `tests/core/engine/test_master_conductor_reauth.py` (9 tests pass)
