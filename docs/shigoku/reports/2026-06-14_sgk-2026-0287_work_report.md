## 3. 判断理由

### `execute_with_replan` 160行未達の原因

`execute_with_replan` は facade として global guard / queue selection / lock / orchestrator 実行 / checkpoint を直接保持する必要があり、これらは service に移動できない。残削減候補:

- batch execution + timeout recovery block (~66行) を facade helper `_execute_batch_with_recovery()` に抽出
- batch creation + queue wait block (~16行) を facade helper `_prepare_next_batch()` に抽出

これらは facade helper 化で約80行削減見込み（178→~98行）。ただし master_conductor.py 総行数は変わらない。

### 行数純減未達の原因

本タスクの抽出先は facade helper method に集中し、code は master_conductor.py 内に留まった。service 側へ移動したのは Plan/Decision の純粋ロジックのみ（~60行相当）。総行数削減には ReconPipeline adapter 抽出や AgentFactory fallback cookie 管理抽出など、他ファイルへの外出しが必要。

### スコープ縮小の有無

なし。計画書 §3.2 の全5抽出単位を実装した。Recon adapter / dispatch 追加抽出は non-goal として計画書に明記済み。
