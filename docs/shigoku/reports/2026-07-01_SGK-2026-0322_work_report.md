# 作業報告書: SGK-2026-0322

## 実装内容

計画書に従い、7つの実装Unitを完了した。

### Unit 1: ReconState P1a schema contract
- `ReconState` に `parallel_task_progress: dict[str, dict[str, Any]]` フィールドを追加
- `_build_serializable_dict()` に serialization を追加
- `_load_v0()` / `_load_v1()` で欠落時に `{}` で補完
- `PARALLEL_TASK_CHECKPOINT_VERSION = 1` 定数を追加
- テスト: `parallel_task_progress` round-trip, v1/v0 load fallback

### Unit 2: Checkpoint update helper and artifact normalization
- `ReconState.update_parallel_task_progress()` メソッドを追加
  - 状態遷移: running → completed/failed/skipped/rerun_required
  - `attempt_count` 増分（初回 running 時、または再実行時）
  - `started_at` / `updated_at` / `completed_at` の自動設定
  - `artifact_refs` の正規化（path/kind/size/mtime/exists）
  - `error_summary` / `resume_reason` の記録
- テスト: running→completed, artifact merge, failure recording, skipped, attempt increment

### Unit 3: ParallelTasks checkpoint hooks
- `visual_recon()` に `state=None` 任意引数を追加（後方互換）
- 全4タスク（full_port, visual, permutation, dead_sub）に checkpoint hook を追加
  - 終端状態（skipped/completed/failed）の記録
  - 成果物 artifact_refs の記録
  - running 記録と save() は pipeline wrapper 側に寄せた
- テスト: skip on no_subs, completed with artifact_refs, visual state=None compat, permutation already_executed skip, dead_sub completed

### Unit 4: Pipeline checkpoint persistence wrapper
- `run_parallel_tasks()` 内に `_run_checkpointed()` wrapper を追加
  - 実行前に running 記録 + checkpoint save
  - 例外発生時に failed 記録 + checkpoint save
  - 終了後に checkpoint save（task body が記録した artifact_refs を永続化）
  - `asyncio.gather(return_exceptions=True)` の既存挙動を維持
- テスト: checkpoint saved on task completion, exception recorded as failed

### Unit 5: Resume decision and artifact consistency
- `ReconState.get_parallel_task_resume_decision()` メソッドを追加
  - `status == "completed"` かつ artifact 整合時のみ `skip` を返す
  - 欠損 artifact / zero-byte / 古い mtime / size 不一致 / checkpoint_version 不一致 → `rerun_required`
  - 未完了 status / エントリなし → `run`
- テスト: skip valid, rerun missing/zero-byte/version-mismatch/old-artifact, run not-completed/no-entry

### Unit 6: Wire resume decision into run_parallel_tasks
- `run_parallel_tasks()` が task list 構築前に resume decision を評価
- `skip` タスクは coroutine を作らず `resume_reason` を progress に追記
- `rerun_required` タスクは `attempt_count` を増やして再実行
- chained task（full_port → dead_sub）の resume 判定は独立評価
- テスト: skip valid completed, rerun missing artifact, dead_sub after full_port skip

### Unit 7: Regression and integration verification

## 既知のリスク

1. `ReconState` の schema_version は 1 のまま。`parallel_task_progress` の欠落は load fallback で補完されるため後方互換は維持されるが、将来 schema v2 への移行時に checkpoint_version mismatch の扱いを再検討する必要あり。
2. 並行タスクの artifact_refs は `update_parallel_task_progress()` のファイルシステム解決に依存する。ファイルシステムの遅延や権限問題で false negative が発生する可能性あり。
3. `visual_recon` の `state` 任意引数は後方互換だが、全呼び出し元が state を渡すように移行することを推奨。
