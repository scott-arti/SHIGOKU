## 3. 判断理由

### ReconPipeline adapter 見送り
`ReconPipeline` の MC 依存が深く（`self.mc` 保持、`auth/cookie/context`、`_add_tasks`、`llm_client` アクセス）、adapter 導入は `ReconPipeline` 契約再設計に相当するため別スパイク扱い。

### execute_with_replan 軽量抽出
`main.py` と `interactive_bridge.py` からの production caller が存在するため、深層抽出（batch recovery の service 化）を見送り。batch size + timeout 計算のみ pure function 化。
