# Ollama廃止とLLM設定統一 作業完了報告書

## 達成ゴール

- [x] Ollama 依存をやめ、APIベースの複数LLMを用途やサブエージェントごとに使い分けられる設計にした
- [x] Ollama を安価なAPIモデル（deepseek/deepseek-v4-flash）へ置き換えた
- [x] LLM設定を `config/shigoku.yaml` に一元化し、role/model/provider/api_key_env/timeout/temperature/fallback を設定ファイルだけで調整可能にした
- [x] Swarm、Worker、Custom AI、Conductor、RAG圧縮、tool output analysis などの用途別に最適な LLM role を割り当てた
- [x] 各 role の system prompt を `src/prompts/roles/` に一元管理し、コード内の直書き system prompt を段階的にテンプレート化した
- [x] `TaskComplexityClassifier` による暗黙分岐を廃止し、役割別の明示設定（role）へ寄せた

## 実装サマリー

### Phase 1: 設定Schema + Validator
- `config/shigoku.yaml` に `llm.schema_version`, `providers`, `profiles`, `roles` ブロック追加
- `src/core/config/settings.py` に `LLMProviderSettings`, `LLMProfileSettings`, `LLMRoleSettings`, `LLMSettings` 追加
- 設定優先順位: `explicit init args > env override > config/shigoku.yaml > defaults`
- Validator: fallback自己循環検出、api_key_env未設定エラー、prompt template欠落エラー、default_role存在検証

### Phase 2: Ollama直依存除去
- `LLMClient` を role-aware に変更（`role` パラメータ追加、`LLMRoleResolver` で解決）
- MicroAgent、RAG compress_context、proxy_log_analyzer、log_translator、VisualRecon の Ollama 直叩きを除去
- `LocalLLMProvider` / `TaskComplexityClassifier` を非推奨化
- 全 "ollama/qwen3.5:latest" fallback文字列を deepseek に置換
- 通常実行パスでの Ollama API/CLI呼び出し: **0件**

### Phase 3: Swarm/Worker/Custom AI role移行
- base_manager、master_conductor、general_agent、parallel_tasks の `model_lightweight` → `role="swarm_manager"/"specialist_light"`
- injection agents (smart_sqli/xss/cmd_ssrf/lfi, actor_critic) → role個別化
- swarm specialists (discovery/logic/auth) → role-based
- `base.py` / `runner.py` / `factory.py` の `LLMClient(model=...)` → role-based
- base_manager の turn別 model 選択 → `_select_role_for_turn()` に変更

### Phase 4: prompt統一
- `LLMClient._maybe_inject_system_prompt()` で role の template を自動注入
- inline system prompt を主要3箇所から除去（rag, proxy_log_analyzer, general_agent）
- 注入 agent用に個別 role 作成（xss/sqli/lfi/cmd_ssrf/actor_critic/recipe_generator）
- `vision_analysis` role 有効化（`optional: true` でAPIキー遅延検証）
- cache key に role 文脈を含めて role 間衝突を防止
- `config/shigoku.yaml.example` を本体設定に同期
- `docker-compose.yml` の Ollama env 除去
