## 4. 残リスク・気づき

### injection 系 read_only 化の観測結果
- `SmartSQLiHunter`/`SmartXSSHunter` 等の injection specialist 群は Phase 0 で `rate_limited`（read_only+parallel_safe=true）に分類。Phase 2 `CATEGORY_TO_LANE` の `attack_inject→mutating` とは不一致（`lane_disagreement=true` で記録）。
- **injection swarm 全体**としては `ActorCriticFuzzer`（aggressive_exclusive）が最制約となり `aggressive_exclusive` lane で記録。
- Phase 5 Go 前に injection 系の read_only 候補の個別レビューが必要。

### disagreement 件数
- `attack_inject` / `attack_auth` カテゴリで lane_disagreement=true（Phase 0: aggressive_exclusive vs Phase 2: mutating）

### MC wiring risk
- shadow decision 計算は `try/except` で包み、失敗時は実行に影響しない
- `_lane_policy`/`_mutex_policy` の lazy 初期化に失敗しても `None` のまま継続
- 実行順・findings・request 数への副作用はテストで検証済み
