# SGK-2026-0279 作業完了報告書

## 1. 要約
- `ToolRunner` に demo provider の注入点を追加し、`SHIGOKU_DEV_MODE=true` 時の mock 出力責務を `src/recon/demo/provider.py` へ移した。
- `ReconPipeline` の `fetch_resolvers()` と whatweb fixture 保存を provider 経由に寄せ、DEV_MODE のファイル副作用を本番経路から分離した。
- 既存の `pipeline.runner.dev_mode = True` を使うテスト互換を維持したまま、DEV_MODE の経路選択と fixture 生成を targeted test で固定した。

## 2. 変更内容
- `src/recon/demo/provider.py`
  - Recon 用の `ReconDemoProvider` を新規作成
  - subfinder / amass / assetfinder / httpx / katana / whatweb の deterministic mock 出力を実装
  - resolver fixture 保存、whatweb fixture 保存 helper を実装
- `src/recon/tool_runner.py`
  - `demo_provider` 注入を追加
  - `check_tools()` / `is_tool_available()` / `run()` が DEV_MODE 時に provider へ委譲するよう変更
  - `SHIGOKU_DEV_MODE` 未設定時は production、`true` 時は demo となる既存判定を維持
- `src/recon/pipeline.py`
  - `fetch_resolvers()` を provider 経由で fixture 保存するよう変更
  - whatweb の `--log-json` fixture 保存を provider 経由へ変更
- `tests/unit/test_robustness_phase4.py`
  - demo provider 委譲のユニットテスト追加
  - 環境変数による DEV_MODE 経路選択テスト追加
- `tests/recon/test_step3_livecheck.py`
  - resolver fixture と whatweb fixture が provider 経由で保存される targeted test を追加

## 3. 設計判断
- `ToolRunner` には subprocess 実行 abstraction を残し、DEV_MODE 固有の mock 文字列生成は provider に隔離した。
- `pipeline.py` の DEV_MODE 分岐は、ファイル副作用を持つ箇所から優先的に剥がし、互換性のため `runner.dev_mode` 切替 API 自体は残した。
- `proxy_integration.py` については `BurpIntegration` 実装が既に削除済みであり、今回のコード変更は不要と判断した。
