## 1. 反実仮想思考 (Counterfactual Thinking) in CoT

- **導入日**: 2026-01-01
- **対象ファイル**: `src/prompts.py` (`COT_INSTRUCTION`)
- **ステータス**: 🧪 実験中

### 概要

エージェントの Chain of Thought (CoT) プロンプトに「反実仮想思考」セクションを追加。
エージェントが「このパラメータは安全だ」と安易に結論を出すのを防ぎ、隠れた脆弱性やビジネスロジックの欠陥を探り当てることを目的とする。

### 追加された思考ステップ

```
4. **Counterfactual (反実仮想)**: もし期待した脆弱性が存在しなかった場合:
   - 他に見落としている入力ポイント（隠しパラメータ、HTTP Header、Cookie、ファイルアップロード）はないか？
   - 異なるユーザー権限（未認証/低権限/高権限）での振る舞いは確認したか？
   - 処理のタイミングやシーケンスを変えた場合、異常動作はないか？（レースコンディション）
   - ビジネスロジック上の前提（価格、数量、状態遷移）を破壊できないか？
```

### 期待される効果

- 確証バイアスの軽減
- ビジネスロジック脆弱性の発見率向上
- 表面的なテストからの深層テストへの誘導

### 懸念事項

- LLM へのトークン消費増加（コスト・レイテンシ）
- 指示の複雑化によるエージェントの遵守率低下

### 効果検証方法（リリース後 TODO）

1. **A/B テスト**: 同一ターゲットに対し、反実仮想思考あり/なしで診断を実行し、発見数を比較
2. **定性評価**: エージェントの出力ログを確認し、Counterfactual セクションが実際に活用されているかを確認
3. **パフォーマンス計測**: 診断完了までの時間・トークン消費量を比較

### 無効化方法

`src/prompts.py` の `COT_INSTRUCTION` 内の Counterfactual セクション（4 項目目）をコメントアウトする。

```python
# 4. **Counterfactual (反実仮想)**: ...
#    - 他に見落としている...
#    - 異なるユーザー権限...
#    - 処理のタイミング...
#    - ビジネスロジック上の前提...
```

## 将来の実験的機能候補

以下はまだ実装されていないが、検討中のアイデア：

- **Adversarial Self-Critique**: エージェントが自身の結論に対して反論を生成する
- **Hypothetical Attack Tree**: 発見した情報から仮想的な攻撃ツリーを構築し、最も有望な経路を選択
- **Confidence Scoring**: 各発見に対して確信度スコアを付与し、低確信度の項目は再検証を促す
