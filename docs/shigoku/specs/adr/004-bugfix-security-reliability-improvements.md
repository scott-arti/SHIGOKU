# 004. セキュリティと信頼性向上のためのバグ修正設計判断

| ステータス | 承認済                                 |
| ---------- | -------------------------------------- |
| 日付       | 2026-01-05                             |
| 決定者     | Self-Reflective Agent                  |
| 関連       | [Bug Report](../../reports/bug_report.md) |

## 背景 (Context)

包括的なバグ分析の結果、セキュリティ（脆弱性検出ロジックと防御機構）およびシステムの信頼性に関する重大な欠陥が 5 点発見された。特に重大だったのは、Human-in-the-Loop (HITL) における Fail-Open の挙動と、IDOR 脆弱性検出における単純な文字列比較による誤検知・見逃しの問題である。これらのバグ修正にあたり、システム全体のセキュリティポリシーと観測可能性（Observability）に関わる重要な設計判断を行う必要があった。

## 判断 (Decision)

以下の 3 つの主要な設計変更を採用する決定を行った。

### 1. Fail-Closed（安全側への倒し）原則の HITL への適用

- **変更前**: HITL コールバック実行中に例外が発生した場合、デフォルトで `True`（承認）を返し、プロセスを続行していた（Fail-Open）。
- **変更後**: HITL コールバック実行中に例外が発生した場合、ログレベル `CRITICAL` でエラーを記録し、即座に `False`（拒否/停止）を返す（Fail-Closed）。
- **理由**: セキュリティツールにおいて、監視機構の故障が攻撃実行の許可につながってはならない。安全性を最優先する「Fail-Closed」原則を適用する。

### 2. PII ベースの意味的差異検出による IDOR 判定

- **変更前**: 単純な文字列比較 (`!=`) と最低文字数制限のみで IDOR 脆弱性の有無を判定していた。
- **変更後**: 以下のステップによる意味的差異検出を導入した。
  1.  **動的コンテンツ除去**: タイムスタンプ、CSRF トークン、ノンスなどを正規表現で除去。
  2.  **PII キーワード検出**: 差分の中に PII (Email, Phone, SSN 等) に関連するキーワードが含まれるかをチェック。
  3.  **ハイブリッド判定**: PII が含まれる場合は即座に脆弱性と判定。含まれない場合でも、クリーニング後の差分が一定長（20 文字）以上であれば「有意な差分」として検出。
- **理由**: タイムスタンプのみの変更による誤検知（False Positive）を排除しつつ、実際に個人情報が漏洩しているケース（True Positive）を確実に検出するため。

### 3. エスカレーション付きエラーログ

- **変更前**: `requests.RequestException` 発生時、例外を捕捉して単に `continue` するのみで、ログ出力がなかった（Silent Failure）。
- **変更後**: URL ごとにエラー回数を追跡するステートフルなロギング機構を導入。
  - 1〜4 回目: `WARNING` レベルでログ出力。
  - 5 回目以降: `ERROR` レベルにエスカレーションしてログ出力。
- **理由**: 一過性のネットワークエラーによるログ汚染を防ぎつつ、永続的な接続障害や WAF によるブロックなどの異常事態を検知可能にするため。

## 結果 (Consequences)

### ポジティブな影響

- **安全性向上**: HITL システムの故障時でも、意図しない攻撃リクエストが送信されるリスクが排除された。
- **検知精度向上**: IDOR スキャンの信頼性が向上し、運用者のトリアージ負荷が軽減される。
- **運用性向上**: 頻発するネットワークエラーが可視化され、トラブルシューティングが容易になった。
- **堅牢性向上**: 型安全性の修正により、予期せぬ入力値によるクラッシュが防止された。

### ネカティブな影響・リスク

- **可用性への影響**: HITL コールバックにバグがある場合、正常な承認プロセスもブロックされる可能性がある（セキュリティトレードオフとして許容）。
- **処理コスト**: PII 検出のための文字列処理が追加されたため、スキャン時の CPU 負荷がわずかに増加する。
